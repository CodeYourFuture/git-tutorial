Using GitHub

"Forking" a project

Forking a project copies it to your GitHub account. In a way, it is equivalent to cloning it.

Why "forking"?

Forking gives you the ability to work on the project and modify it according to your needs without interfering with the original project.

This is useful if you want to adapt it for something specific, if the original project is no longer active, or... if you want to contribute back to them

Contributing!

The ability to contribute is the most important feature of GitHub. Different people can work on different aspects of a same project, working at their own speed, and then they can deliver their changes or improvements

This is done using Pull Requests

Pull Requests

Allows the receiving project to review the changes before incorporating them

Allows automatically integrating these changes

Good practices

Keep your master branch pointing to the original project's master branch

Work on feature branches

Submit Pull Requests from feature branches

Keep feature branches up to date
