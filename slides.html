Introduction to Git

The problem of changes

Option 0: do nothing

You have a document or project, you modify it, and you save the latest version.

Option 1: keep copies

  • Document.doc
  • Document_20161111.doc
  • Document_May2013.doc

Option 2: use a version control system

Use a tool that keeps track of all the changes in the document or project

What is Git?

A distributed version control system

A version control system is a tool that allows to track changes performed to the contents of a project

"Distributed" vs. "Centralized"

Distributed
There is no need for a central copy. Each developer has a full clone of the repository with all the project history.
Centralized
There is a single central copy of the project. Changes are submitted to the central copy.

What is GitHub?

GitHub is a web service that specialises in providing hosting for Git projects.

Although Git is distributed, developers can agree on a central server.

Using Git

Contributing to an existing project

First step: getting a copy of the project

$git clone https://github.com/Code-Your-Future/git-tutorial.git

This will copy all the information about the project to your local hard drive

Second step: creating a development branch

$git checkout -b fixes

Typically you will work in branches. Think of a branch as a "virtual space" where you can group similar changes together.

You can have multiple branches, something useful to work on different things simultaneously

Third step: submitting your local changes

$git add new_file.txt changed_file.js

If you modify an existing file or you create a new one, you add it to Git with git add

Fourth step: grouping changes

$git commit -m "Fixed some bug"

A commit groups changes together into a single unit. For example:

  • Fixed typos and spelling errors.
  • Fixed arithmetic errors in animations.js
  • Updated CSS margins for LoginPage

Fifth step: delivering changes

$git push

This will send your changes to the repository you cloned from (if you have permissions).
